繪圖板、數位板支援¶

最常見的使用者求助個案是關於數位板筆壓感應與其他數位板問題。

快捷解決方法¶

備註

套用以下提到的變更操作之後，可能需要重新啟動電腦才會生效。

針對所有運行 Windows 的裝置：

  1. 到設定 ‣ 設定 Krita… ‣ 數位板設定版面更改數位板輸入 API。

    1. WinTab：舊式數位板 API 標準。它支援多於一個數位筆按鍵和較高的筆壓等級數目。如果這能正常運作，沒有特殊理由的話不應切換到 Windows Ink。二合一平板電腦預設使用 Windows Ink。部分硬體廠商有提供 WinTab 驅動程式，但一般要手動安裝。

    2. Windows 8+ 指標輸入 (Windows Ink)：較新的 API 標準。這個 API 只支援 1024 級筆壓。這比較適用於二合一平板電腦。某些較冷門的數位板品牌可能未有提供這個 API。

針對在 Windows 使用外接式數位板裝置（非二合一平板電腦）：

  1. 重新安裝驅動程式（Windows Update 經常會損壞數位板驅動程式的設定，重新安裝會有幫助）。

  2. Wacom 數位板：如果在起筆時出現直線線段，請先嘗試更新驅動程式。這個問題應該已在驅動程式版本 6.3.34-3 中修正。如果這沒有解決問題，請到 Wacom 數位板設定中停用「雙響間距」。

  3. XP-Pen 數位板筆壓不平均：可嘗試在設定 Krita… ‣ 數位板設定中切換至「Windows 8+ 指標輸入 (Windows Ink)」，或到 XP-Pen 驅動程式設定中停用「Windows Ink」。

收集資料¶

  1. 你使用哪個作業系統？

  2. 你使用甚麼品牌和型號的數位板？

  3. 你裝有的數位板驅動程式版本編號是甚麼？

  4. 請收集數位板測試器（設定 Krita… ‣ 數位板設定 ‣ 開啟數位板測試器）中的文字輸出並提供予支援者：如何分享檔案。

  5. 收集更詳細的數位板事件紀錄：

    1. 開啟設定 ‣ 工具面板 ‣ 紀錄檢視器工具面板，確認已勾選該選項。

    2. 在「紀錄檢視器」工具面板中，確認第一個開關按鈕「啟用紀錄」處於按下狀態（即已啟用記錄事件）。

    3. 按下 Ctrl + Shift + T 以啟用數位板事件紀錄。

    4. 畫出數道筆跡（按情況需求，支援者或開發者可要求作出特定的筆跡動作）。

    5. 按下 Ctrl + Shift + T 以停用數位板事件紀錄。

    6. 在「紀錄檢視器」工具面板中點選第三個按鈕以儲存紀錄到檔案。

    7. 將檔案或檔案內容提供予支援者：如何分享檔案。

    使用 Linux 的話可以直接從主控台視窗取得紀錄內容。這樣只需要啟用數位板事件紀錄，可以跳過「紀錄檢視器」工具面板。

給支援者的額外資訊¶

  1. 除了起筆時出現直線線段的問題之外，Wacom 數位板通常都能在各個作業系統上正常運作。

  2. 繪王 (Huion) 數位板在 Windows 和 Linux 上應該能正常運作，但在 macOS 上可能會有問題。

  3. XP-Pen 以及其他品牌的數位板在各個作業系統上都可能會有各種問題。

  4. 有關選擇數位板的建議，針對與 Krita 使用的話，一般來說一塊較便宜的 Wacom 或 Huion 數位板會是較好的選擇（2019 年計）。可參見支援的繪圖板、數位板。

  5. 對於 XP-Pen 數位板出現的問題，這裡的指示可能會有幫助。

動畫¶

與動畫繪製輸出相關的問題可以是各式各樣的。首要步驟是查出問題根源是在 Krita 內部還是 FFmpeg 方面（Krita 先輸出所有影格，然後 FFmpeg 用這影像序列產生影片檔案）。為找出答案，請指示使用者將動畫繪製匯出為「影像序列」。如影像序列輸出正確無誤，這表示是 FFmpeg 或者繪製選項（後者較常見）出現問題。如影像序列輸出不正確，有機會是一般選項出錯（例如沒有繪製所有影格），又或者是 Krita 有程式錯誤。

備註

如使用者開啟「紀錄檢視器」工具面板並啟用紀錄，然後嘗試繪製動畫影片輸出，Krita 會將整句 ffmpeg 命令列輸出到「紀錄檢視器」之內供調查用。

在繪製動畫的輸出目錄內會有一個名為「log_encode.log」的紀錄檔。這個檔案包含了對調查問題有幫肋的資訊（請參見如何分享檔案）。

洋蔥皮相關問題¶

絕大部份與洋蔥皮相關的問題都是使用者操作不當，而不是程式的問題。儘管如此，找出問題發生的原因，還有是指導使用者正確使用洋蔥皮，都是提供技術支援要做的工作。

軟體崩潰¶

如遇到程式強制終止的情況，應嘗試判斷這是否已知的問題。假如不是，應指示使用者到 Bugzilla 建立一個缺陷報告，或者由支援者代為建立。缺陷報告內應含以下資訊：

  1. 發生了甚麼事？程式強制終止前那一刻在做甚麼？

  2. 能夠重現這個問題（使其重複發生）嗎？如可，請提供一系列可以重現這個問題的步驟。

  3. Backtrace（當機紀錄）——給 Windows 使用者的指示在這裡：Dr. MinGW 除錯工具，而偵錯符號封裝的下載連結可在該版本的發佈新聞內找到，亦可直接在 https://download.kde.org/stable/krita 裡面找到。

其他可能有快捷解決方法的問題¶

  1. 如使用者遇到任何未聽聞過的奇怪的問題，請要求該使用者先嘗試重設 Krita 的設定選項。

  2. 如有 Windows 的使用者遇到任何與畫布顯示有關的問題，請要求該使用者嘗試更改設定 ‣ 設定 Krita… ‣ 顯示內畫布圖形加速的「繪製器」設定。

    備註

    在任何情況下都不應建議使用者停用「畫布圖形加速」來改善效能。

給支援者的忠告¶

  1. 如果未能清楚理解求助者的提問，應請求闡明一下。要求提供畫面截圖或錄影是不錯的做法。

  2. 如果不知道解決方法，但知道調查問題需要甚麼額外資訊的話，可以先向求助者查詢。這樣可以幫助其他知道解決方法的支援者更快能夠提供進一步支援。這尤其適用於與數位板有關的問題。

  3. 如果不知道答案或解決方法而問題看來被其他支援者忽視，不妨到 Krita 的 IRC 聊天室提問（Libera.Chat 上的 #krita）：Krita 的社群。

  4. 向使用者清楚解釋他應採取的步驟。例如要修改某項設定的話，應完整列出每個步驟要點選的按鈕、選單與分頁。

  5. 不要寫設定 ‣ 設定 Krita…，只需要寫設定 Krita——這個選單項目容易找到，而且這樣寫不會使 macOS 使用者產生混淆（因為在 macOS 上要經由Krita ‣ 偏好設定...才能開啟設定）。

  6. 如要求求助者提供影像，可建議 Imgur 或 Pasteboard（如何分享檔案），否則 Reddit 使用者可能會用該影像發表一篇新貼文而非從原本的討論串連結到該影像。

  7. 快捷回答：如本說明文件內有相關內容，可直接連結到說明文件中相應的頁面。點擊章節標題旁的連結圖示即可複製連結，以提供予求助者作參考。

如何分享檔案¶

  • 影像（例如畫面截圖）：Imgur [*]、Pasteboard

  • 純文字：Pastebin [*]、BPaste、paste.ubuntu.org.cn、CentOS 的 Pastebin 服務、KDE Snippets（需要 KDE Identity）

  • .kra 以及其他檔案格式：電子郵件。或可以經 Linux 提供的 base64 命令列工具編碼並放到純文字分享服務，然後用相同的工具解碼。

    注意

    如果要求求助者經純文字分享服務提供紀錄或其他資料，請確保這些資料的保存時長足夠讓你存取並下載——例如 bpaste.net 網站預設只會將檔案保存一天，而最長只可延長到保存一個星期。

    被封鎖的網站

    如果使用者被某防火長城包圍（例如中國大陸），上述帶有 [*] 標記的網站很大機會已被封鎖。請使用其他替代網站。
