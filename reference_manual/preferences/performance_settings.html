「效能」設定¶

Krita 作為一個數位繪圖軟體，它需要處理很多的資料，例如您使用的筆刷、您所選擇的色彩，但主要是影像中的每個像素。因此如何讓 Krita 組織它所儲存資料的方式會影響 Krita 呈現的效能與速度。就像一個條理分明的工作室可以提高現實中藝術家繪畫的效率。

這些偏好選項允許您組織並設定 Krita 的效能，例如記憶體、CPU、快取這些與效能有密切關係的項目。這頁面內所有項目經變更後需要重新啟動 Krita 才會生效。

記憶體¶

「隨機存取記憶體」（RAM 或 Random Access Memory）以下簡稱「記憶體」，是一種容量小但讀寫速度很快的暫時容器，關機後資料就會消失；硬碟則用作永久儲存資料的容器，相對記憶體而言容量很大但是讀寫速度很慢。而 CPU 運算時會先將資料由硬碟讀進記憶體中以加快處裡速度，由硬碟抄寫進記憶體與清空記憶體會使用一些時間。因此恰當分配與運用電腦有限的記憶體，可提高 :program: Krita 的效能 。

您可經由這些設定選擇要將多少資源分配給 Krita 使用，並且每次 Krita 啟動都會使用這些設定。因此這頁面內所有設定經變更後都需要重新啟動 Krita 才會生效。

記憶體上限

這是指定給 Krita 啟動後可用的記憶體大小上限。這裡以百分比與右側實際大小兩種方式顯示，因此您可以精確指定 Krita 不要使用佔用超過上限的記憶體，讓電腦可以保持背景執行網路瀏覽器或播放您的音樂。

記憶體保留上限

這是高級使用者選項，這功能是允許 Krita 將資料放在記憶體中一個虛擬儲存區塊。就好像藝術家在畫布上有個標準位置一樣，也會因為給予 Krita 一個專有的資料存取區域提高資料讀寫與搜尋效能，也避免因為記憶體碎片問題遺失資料。

增加這個區塊的大小等同於有更多空間儲存這些資料，但也會因資料無法放滿整個空間顯得浪費，設定得太高也會造成某些資料 Krita 不知要放在哪裡，反而因此效能降低。

另一方面，若不給予或設定的數值太小，也會因此花更多時間在尋找存取的位置。這樣也會影響 Krita 的執行效能。

建議值是設定為您常用影像單一圖層所需記憶體的大小。例如一個 3000 x 3000 像素的 8bit RGBA 影像大概需要36 MiB。

因為 Krita 必須在啟動時才能向系統要求取得記憶體的分配，在做這些更改之後需重新啟動 Krita 設定才會生效。

在 4.4 版本開始棄用: 此選項在 Krita 4.4 版之後已經移除，它現在由程式自動控制，使用者不需再做設定。

復原資料置換前上限

Krita 的復原資料必須保存在記憶體中，超出這個數值限制的舊復原資料會直接寫入檔案。如果您的記憶體不足，可以降低這個數值將更多記憶體保留給目前的作業，但若這個數值設定太低，經過多次的復原操作時可能會造成 Krita 緩慢問題。變更這個數值後需重新啟動 Krita 才能生效。

置換檔案大小¶

檔案大小限制

指定 Krita 可以使用的交換檔空間大小上限。若資料同時超過這個記憶體與交換檔的大小上限，可能會造成 Krita 軟體凍結。

置換檔案位置

指定 Krita 交換檔保存於硬碟的位置。這個交換檔位置對於 Krita 性能也會有影響，例如放在固態硬碟 (SSD) 會比傳統機械硬碟 (HDD) 來得快。有些人會使用 USB 外接碟作為交換檔空間，但前提是這 USB 外接碟性能要夠好，且也受到 USB 連接埠速率所影響，例如 USB 3.0 會比 USB 2.0 快。若 USB 外接碟只支援到 2.0 ，那就算接到電腦 USB 3.0 的連接埠，那依然也只能用 USB 2.0 速率。

進階¶

多執行緒¶

從 4.0 版開始，Krita 支援了動畫快取多執行緒處理，並且像素筆刷引擎的筆尖圖案功能都已完成多執行緒的最佳化。

CPU 限制

指定 Krita 可使用的執行緒數量上限。

動畫繪製平行副本上限

Krita 在繪製動畫時，會先將影像複製為多個複本，對大值由 CPU 核心數決定，然後在盡可能多的 CPU 執行緒中平行同時處理以提高速度。這表示每一份複本都需要佔用額外的記憶體，如果處理的動畫很龐大又複雜時，為了不耗盡記憶體，這時有必要限制處理動畫使用的執行緒數量。

其它¶

繪畫模式每秒影格數限制。

降低繪製時畫布更新率。這讓 Krita 可以保留處理其他事情的資源，因為有些人很在意筆刷更新頻率，因此加入這個選項讓用戶自己設置。

啟用除錯用的OpenGL影格速率記錄

啟用後將在畫布上顯示影格速率。

啟用供除錯用的筆刷繪製速度記錄。

啟用後將顯示一組數字，顯示最後一個畫筆描繪的速度。

停用AVX 向量最佳化（給 AMD CPU）

向量最佳化是一種要求 CPU 進行數學運算的特殊方式，相關指令集包括 SIMD 和 AVX 等。這些最佳化可以讓 Krita 繪製時更快。假如您 Windows 電腦使用的是比較老舊的 AMD CPU ，當執行 Krita 繪製發生一些奇怪的狀況時，可以將這個項目打鉤停用。

程式進度報告

選用這項目後，在進行需要長時間處理繁重工作（例如過濾器或大型筆刷）時，Krita 會在狀態欄顯示進度條表明當前狀態，欄位旁邊的紅色停止按鈕可以中止這個顯示。這功能預設開啟，您可以在這裡將它關閉。

程式效能記錄

啟用這個項目時，Krita 會將程式性能記錄寫入到 工作目錄 的 Log 檔案中。這個 工作目錄 也是自動儲存的目錄。

如果影像尚未命名， Linux 環境下的日誌會寫入到 $HOME 資料夾，而 Windows 環境則是 %TEMP% 資料夾。

動畫快取¶

在 4.1 版本新加入.

動畫快取是動畫幀在記憶體中所佔用的空間，是將預先計算好的影像放在快取中。

假設我們要播放一個 25 FPS 的動畫，這表示電腦需每秒繪製 25 張影像。影片播放軟體可以為此做大量最佳化以提高效率，但 Krita 必須允許使用者編輯影像，因此無法像影片播放軟體那樣做出最佳化。

儘管如此，動畫師也必須看到製作中的是怎樣的動畫。為了正確達到這個要求，因此我們需要決定 Krita 要如何在動畫師進行更改後重新生成快取。幸運的是我們有很多不同選擇可以做到這個要求，最好的解決方案就是依序動畫的特性在電腦配備的效能上進行調整。在這個選項中您可以自定義生成快取的方式和場合。

快取存放空間後端（快取空間的位置）¶

使用記憶體

選用這個項目時，動畫快取將會儲存在記憶體中，且不做任何限制，這也是 4.1 版之前預設的處理方式。但這選項對於系統記憶體需求會比較多，若您製作的動畫經常都需全螢幕解析度為 6K 每秒 25fps ，那麼您的電腦最好有安裝 64GB （或以上）的記憶體。如果您電腦沒有這麼多記憶體，那麼最好*不要*使用此選項，同時也請考慮降低動畫的解析度。

警告

請確保您的電腦有足夠的實體記憶體。實體記憶體必須*大於*您在 :guilabel:一般中所指定的容量，否則您可能會遇到系統凍結鎖死的狀況。

  • 每秒 25 FPS 的FullHD （全高清1080P）動畫需要 200MiB 記憶體。

  • 每秒 25 FPS 的 4K UltraHD （超高清 4K）動畫需要 800MiB 記憶體。

使用硬碟

使用此選項時，動畫幀會預先壓縮並與記憶體交換檔儲存在同一個硬碟的目錄中。此選項需使用少量實體記憶體以供運作。

因為硬碟的資料傳輸速度比較慢，建議您將 :guilabel:限制快取影格的大小設定為 2500px （像素） 以便讓 Krita 能在 25fps 的速度下播放影片。

快取產生選項¶

限制快取影格大小

如果製作的影像畫質超過此大小限制，則將快取影格製成減少畫質的縮小版，在選用*硬碟*為快取儲存空間時，請確保啟用此選項。因為硬碟讀寫很慢，如果按照完全畫質寫入快取，它很有可能因無法全速播放而出現卡幀的狀況，所以這是用比較低的畫質來換取影片流暢的播放，請放心這個功能並不會影響最後輸出的畫質。

使用重點區域

就技術而言，我們僅需使用視圖中有影像的那部分畫面。啟用此選項並設定數值後， Krita 將以此值為限制，繪製可見畫面外百分比之區域。

啟用在背景產生快取

這個選項可讓您選擇是否在電腦不使用時於背景製作動畫快取，這可以減少快取等待時間，有利於流暢的播放動畫。若您想節省電腦的耗電量那就關閉這個選項。

即時預覽¶

變形工具顯示圖層堆疊中的預覽

啟用此選項後，將顯示已繪製圖層混合模式效果的變形工具預覽。取消鉤選時則浮動顯示預覽。

強制變形工具使用即時預覽

就算在選單中檢視‣ 即時預覽功能未啟用的狀態下，乃強制開啟變形工具的即時預覽。

強制移動工具使用即時預覽

就算在選單中檢視‣ 即時預覽功能未啟用的狀態下，乃強制開啟移動工具的即時預覽。

強制濾鏡使用即時預覽

就算在選單中檢視‣ 即時預覽功能未啟用的狀態下，乃強制開啟filtersl的即時預覽。
