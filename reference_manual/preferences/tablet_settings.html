「數位板」設定¶

數位板

全域筆刷輸入調整曲線：這是您的繪圖板將在 Krita 中使用的全域曲線設定，這裡的設定會影響您繪圖板壓力手感柔軟或偏硬。

透過滑鼠事件讀取右鍵和中間鍵點擊。

某些二合一平板電腦無法得知是否有手寫筆上的側面按鍵。如果您有這樣的設備，可以嘗試啟用這個解決方式。Krita 會嘗試讀取右鍵和中間鍵的點擊，以模擬這些動作來自滑鼠而不是這一台平板電腦。這功能可能會也可能不會在您的設備上正常運作，這取決於平板電腦驅動程式的執行效果。變更這個選項後需要重新啟動 Krita 。

在 4.2 版本新加入.

以數位板驅動程式提供的時間來計算筆刷速度

勾選時是經由驅動程式的時間標記改變 Krita 徒手繪圖筆刷工具防抖動平滑化，不勾選時則是使用計時器的時間標記。

在 5.0 版本新加入.

數位板輸入 API（Krita 3.3 或更高版本）

以下項目僅適用於 Windows 8 及以上作業系統版本。WIndows 8 及以上的版本設備都有兩種數位板軟體連接方式，有些設備則只能在一種連接方式正常使用。變更後需要重新啟動。

使用 WinTab 方式連接

以 WinTab API 方式接收感壓筆的輸入。雖然這是 Krita 3.3 版之前使用的API，但對於大多數 Wacom 數位板使用者建議使用此選項。

Windows 8+ 指標輸入

使用指標輸入的方式接收數位板的輸入。這選項取決於數位板驅動程式是否有 Windows Ink 的支援。這是相對較新的方式，所以仍然被認為是實驗性，不過應該也能正常用於繪畫。若您使用的是 N-Trig 設備（例如微軟的 SurFace 平板電腦），或您的數位板以 WinTab 方式無法正常操作，請改用這個方式試試看。

進階 WinTab 數位板設定¶

使用多台螢幕或使用可以做為螢幕的繪圖平板時，WinTab 可能會讓 Krita 將數位板映射到錯誤的螢幕區域造成畫面偏移，您可以在「映射到自訂區域」手動設定。

使用數位板提供的資訊

使用 WinTab 數位板驅動程式提供的映射資料。

映射到整個虛擬桌面

使用 Windows 提供的資料映射到整個虛擬桌面以及多個螢幕。

映射到自訂區域

手動輸入螢幕大小和偏移數值。如果您使用多螢幕且其它設定還會出現感壓筆尖對不上螢幕游標位置時，可改用這個選項，但您可能需要反覆試驗，直到完全符合為止。

如果您使用雙螢幕，而數位板游標只能在螢幕上半部移動，這時您可能需要輸入兩個螢幕的總寬度和兩倍螢幕高度。

在 4.2 版本新加入: 若需要在 4.2 之前的舊版 Krita 版本中進入這個對話框，您必須執行以下操作：

  1. 將感壓筆移出數位板感應的區域外。

  2. 不要使用數位板的感壓筆，而是使用滑鼠或鍵盤啟動 Krita。

  3. 啟動 Krita 後，按著 Shift 鍵不放。

  4. 現在使用感壓筆碰觸數位板，讓 Krita 進行辨識。

如果手動辨識也無法解決問題，而使用的是 Wacom 數位板，那麼這畫布偏移的問題可能是 Wacom 驅動程式偏好選項設定檔錯誤或損壞，而這設定檔沒有通過重新安裝驅動程式刪除或置換。您可經由下面方式進行修復。

要修復這個問題時，請到 Wacom 下載新版驅動程式與 “Wacom Tablet Preference File Utility” ，執行 “Wacom Tablet Preference File Utility” 清除所有偏好選項，然後移除舊版驅動程式並安裝最新版驅動程式。 這應該能讓 Krita 自動檢測正確的設置。

警告

清除 Wacom 所有設定之後數位板設定會恢復為預設值，因此您需要重新校準和設定。

數位板測試器¶

在 4.1 版本新加入.

這是用於調試數位板書的的特殊功能，當您打開後他包含兩個視窗區域，請以感壓筆在左邊繪圖區繪製，校准結果會在右邊 文字輸出區顯示數位板輸入的資料。

使用數位板感壓筆在繪圖區中繪圖時，會出現顏色軌跡，當您的數位板執行工作時，這裡應該會同時顯示紅色與藍色線。

紅色代表滑鼠事件。滑鼠事件是 Krita 可以接受的最基本事件，但是滑鼠事件僅有粗略的座標沒有壓力感應。

藍色代表數位板事件。僅在 Krita 可以正常使用數位板時才會顯示。這包含精確的座標與壓力感應數據。

重要

如果在繪圖區繪圖時沒有顯示藍線，這表示 Krita 無法正常使用您的數位板。請查看數位繪圖板頁面取得更多詳細資料。

當您以感壓筆在繪圖區繪製一條線時，右邊文字區域會顯示各種輸出。這輸出的文字內容可以附加到請求協助或錯誤報告中，以讓我們瞭解發生什麼問題。
