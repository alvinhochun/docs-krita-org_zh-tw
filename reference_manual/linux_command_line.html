命令列及參數¶

Krita 是 Linux 原生的應用程式之一，因此它也能使用命令列模式直接處理影像。這項功能原本無法在 Wondows 和 macOS 作業系統使用，但自從 Krita 3.3 版開始，他們也能在命令列模式處理影像了！

這功能主要用在 bash 或 shell 指令，例如一次性批次處理將大量 KRA 檔案轉換為 PNG 檔案。

匯出¶

你可以在終端機模式使用以下命令和參數快速匯出檔案與轉換檔案格式：

krita 匯入檔案名稱 --export --export-filename exportfilename

importfilename

將「importfilename」更換為您要匯入的檔案名稱。

--export¶

匯出檔案時使用的選項。

--export-filename <filename>¶

請將「Export filename」改為匯出時指定的檔案名稱。

exportfilename

將這段文字更換為輸出時的檔案名稱。使用不同延伸檔名可轉存不同檔案格式。

範例：

krita file.png --export --export-filename final.jpg

這一行命令會將 file.png 轉存為 file.jpg。

--export-sequence¶

在 4.2 版本新加入.

將動畫導出到指定的檔案名稱並退出。

如果 KRA 檔案裡沒有動畫，則此命令會在命令列打出 "This file has no animation." 的錯誤訊息之後什麼也不會做。

krita --export-sequence --export-filename file.png test.kra

這一行命令是使用 test.kra 檔案的動畫，以 --export-sequence 指令確定序列，並使用 --export-filename (file.png) 指令來匯出檔案名稱為 file 的 PNG 格式檔案。

匯出為 PDF¶

PDF 格式的匯出稍微不同，這時須使用 --export-pdf 選項。

krita file.png --export-pdf --export-filename final.pdf

這一行指令會將 file.png 檔案轉存成一個叫做 final.pdf 的 PDF 檔案。

警告

因為得到的結果並不正確，這個轉存 PDF 的命令參數已經在 3.1 版移除。

以自訂顯示器 DPI 開啟¶

以指定的螢幕解析度（DPI）來啟動 Krita。

--dpi <dpiX,dpiY>¶

以指定的螢幕解析度（DPI）來啟動 Krita。

例如：

krita --dpi <72,72>

開啟範本¶

啟動 Krita 定自動打開指定的範本。您可以使用這種方式建立桌面捷徑，以這種方式來在啟動 Krita 時自動打開指定的範本，這樣就可以馬上進行工作了！

krita --template templatename.desktop

--template templatename.desktop¶

用來指定範本的參數。

所有範本都以 .desktop 延伸檔名保存。您可以在`` .local/share/krita/template`` 或 Krita 安裝目錄中找到範本。

krita --template BD-EuroTemplate.desktop

這條指令會在 Krita 啟動後打開歐式 BD 漫畫範本。

krita --template BD-EuroTemplate.desktop BD-EuroTemplate.desktop

這條指令會在啟動 Krita 後打開兩個獨立的歐式 BD 漫畫範本。

啟動參數¶

在 3.3 版本新加入:

--nosplash¶

啟動 Krita 時不顯示啟動畫面。

--canvasonly¶

以「畫布獨佔模式」啟動 Krita。

--fullscreen¶

以全螢幕模式啟動 Krita。

--workspace Workspace¶

以指定的工作空間啟動 Krita。例如...

krita --workspace Animation

啟動 Krita 在動畫工作空間。

--file-layer <filename>¶

啟動 Krita ，並將 filename 為檔名的影像添加一個檔案圖層。使用這個指令參數時，您必須同時打開一個影像，或者使用 new-image 指令新建一個影像。

範例：

krita file.kra --file-layer image.png

krita --new-image RGBA,U8,1000,1000 --file-layer image.jpg

如果系統中已有一個 Krita 正在執行，而且只允許一個 Krita 實例執行是停用的狀態，那麼這個指令會將檔案圖層添加到執行中 Krita 檔前的影像中。

例如： krita --file-layer image.png
