設定 Krita 以使用眼球追蹤技術的示例¶

注意

這不是參考文件。 只是一個用戶的體驗。 當使用不同的眼球追蹤設備或不同的控制軟體時，該資訊可能不適用。

眼球追蹤設備越來越便宜，並且它們正在進入更多電腦設備與流程途徑。 雖然這些設備的使用者各式各樣，但我們將主要關注身體殘疾並且只能用眼睛與電腦交流互動的使用者。

如果您還沒有類似案例的經驗，那在開始之前，需要了解以下幾點：

  • 眼球追蹤需要正確校準，使指標非常靠近使用者正在註視的焦點。 要實現這個可能很困難，特別是如果眼球追蹤相對於與用者的定位在不同對話框之間固定時。

  • 由於控制不准確，因此幾乎不可能點擊屏幕上的小區域，例如小按鈕或菜單項。 屏幕的角落和邊緣也可能難以觸及。 您也不希望將界面元素彼此靠近，因為這會增加意外選擇錯誤元素的機會。

  • 單點擊、雙點擊、右鍵單點擊、拖放等鼠標操作都需要在控制設備的軟體中以切換模式的形式進行額外的設定。 目的是減少這些切換的操作次數，以便創作工作不會經常中斷。

  • 切換模式不會自動啟動操作。 為此，需要採取額外的動作。 在範例中，這個動作是“駐停”。 例如，要啟動一個工具，切換到左側雙擊模式，然後停留在工具的圖標上雙擊。 調整停留時間是一個重要的權衡：更短的停留時間允許更快的工作，但也更容易出錯。

要求¶

除了擁有眼球追蹤設備的*明顯*要求外，您還需要一個控製程式，讓您與設備進行互動。 當獲得設備時，設備原廠可能就會附有這樣的程式，但該程式可能不足以將 Krita 連動一起使用。

這些程式的基本功能之一是模擬滑鼠點擊。 在範例中，該程式提供了一個懸停選單，其中包括用於在滑鼠左鍵/右鍵和單擊/雙擊之間切換模式的大按鈕。 選擇模式後，懸停選單可以折疊起來，為軟體留出更多作畫空間。

為了使它們更易於設定和使用，某些程式僅包含單擊等基本模式。 這對於許多時下流行的應用程式（例如電子郵件代理和瀏覽器）來說已經足夠了，但是對於 Krita，需要拖放模式才能進行繪圖。 如果提供的控制軟體不支持此模式（通常稱為“滑鼠模擬”），您可以聯繫設備製造商尋求幫助，或尋找開源的解決方案。

啟動 Krita¶

基本上，將控製程式設定為左鍵雙擊模式並停留在桌面上的 Krita 圖標上就足以啟動 Krita，但這樣做存在一些問題：

  • 啟動時，Krita 會要求您選擇一個模板。 您可能不想每次都要設定一次，而只想從空白模板開始。

  • 稍後，存檔將需要與文件儲存的對話框互動，這對於眼球追蹤的使用者不是很友善。

解決這些問題的方法是建立並儲存一個空白模板並運行一個腳本程式，該腳本程式將以新名稱複製該模板並將其發送給 Krita。 這是一個用於 Windows 的示例腳本程式碼，它使用時間戳後綴來確保每個文件都有不同的名稱（將 USERNAME 替換為實際用戶名）:

@echo off
for /f "tokens=2 delims==" %%a in ('wmic OS Get localdatetime /value') do set "dt=%%a"
set "YY=%dt:~2,2%" & set "YYYY=%dt:~0,4%" & set "MM=%dt:~4,2%" & set "DD=%dt:~6,2%"
set "HH=%dt:~8,2%" & set "Min=%dt:~10,2%" & set "Sec=%dt:~12,2%"
set "datestamp=%YYYY%%MM%%DD%" & set "timestamp=%HH%%Min%%Sec%"
set "fullstamp=%YYYY%-%MM%-%DD%_%HH%-%Min%-%Sec%"
set filename=USERNAME_%fullstamp%.kra
copy "C:\Users\USERNAME\Pictures\blank.kra" "%filename%"
start "C:\Program Files\Krita (x64)\bin\krita.exe" "%filename%"

雙擊此腳本程式並在與腳本文程式文件相同的資料夾中建立一個新的 Krita 檔案。 由於文件已經有了名稱，因此將避免文件出現儲存的對話框。 結合自動保存，這可以成為有效的儲存方式。

小訣竅

將這些檔案直接存儲在雲服務上會更加安全。

在啟動 Krita 時，您可能還需要處理一些時間問題 :

  • 雙擊 Krita 或腳本程式的圖示並開始加載 Krita 後，停留在圖標上將啟動第二個實例。

  • 同樣，雙擊後，如果另一個視窗意外地被帶到前面，Krita 可能會被遮擋在該視窗的後面。

為了防止這些問題，如果使用者訓練自己注視 ( 因為是眼球追蹤 ) 一些沒有遮擋的地方（例如桌面上的空白區域），直到加載 Krita，這將有所幫助。

排版¶

由於界面元素需要比較大空間，因此必須更有效的使用畫面區域。 在全螢幕模式下運行並關閉些未使用的選單和工具欄是可以採取的手段。 這是我們介面佈局的螢幕截圖：

你應該會想把需要的所有功能放在可以輕鬆按的地方。 對於我們的圖紙，最基本項目是畫筆和顏色。 所以我們決定為這些放置永久介面。

Krita 有許多畫筆，但工具介面必須包含有限數量的筆刷，這樣筆刷圖示才夠大。 我們建議 根據自己的喜好建立筆刷。

有多種選擇顏色的工具，但大多數都不容易使用，因為它們需要相當精準的滑鼠控制。 Python Palette Docker 是最簡單的使用方式，可以從一組預定義的顏色中進行選擇，類似於筆刷。 同樣，類似於畫刷選擇，這將有助於 自定義一組最喜歡的顏色。

一旦對介面佈局感到滿意，另一個可以幫助您的功能是鎖定工具介面。 可能會意外關閉或移動到工作介面。 例如，在拖放模式下，您可能會意外抓住一個工具介面並將其拖到工作區域。 為防止這種情況發生，請將以下設定放入 kritarc file:

LockAllDockerPanels=true

(Check the 常見問題 FAQ for how to find the configuration kritarc file on your system.)

If you're using a hovering mouse control menu like we do, you also have to figure out where to place it when it's collapsed. Put it somewhere where it will be easily accessible but where it will not interfere with Krita. On the screenshot you can see it at the left edge of the screen.

概括¶

概略的工作流程如下所敘述。

To start Krita:

  1. On the desktop, pull up the hovering mouse menu and select left double click mode.

  2. Double click on the new drawing creation script. Look away at some harmless spot until Krita loads.

Drawing with Krita:

  1. Switch to left single click mode.

  2. Select a brush and/or color using the dockers.

  3. Switch to drag and drop mode. You're ready to draw.

  4. Go to the point where you want to start a stroke and dwell until dragging starts (this emulates pressing and holding your finger on the mouse button).

  5. Draw.

  6. When you want to finish the current stroke, dwell at the ending point until you get out of dragging (this emulates lifting your finger from the mouse button).

  7. Repeat the whole process.

Finishing:

  1. Switch to left single click mode.

  2. Click on the button for closing the window.

  3. When warned about unsaved changes, click the button for saving the file.
