一般¶

一般問題

Krita 是甚麼？¶

我們對於 Krita 的發展目標如下：

Krita 是自由和開放原始碼的跨平台應用程式，提供由零開始製作數位藝術作品成品檔案的一站式解決方案。Krita 設計上考慮了頻密、長時間和專注作業而作出最佳化。Krita 明確指定支援的繪畫用途包括插圖、概念藝術、接景製作、紋理貼圖、漫畫和動畫。Krita 由開發者和使用者共同製作，以支援使用者的實際需求和工作流程。Krita 支援開放標準，能夠與其他軟體互相交替使用。

我能否使 Krita 以正體中文顯示介面，而非英文？¶

Krita 會自動根據系統語言設定來選擇使用者介面語言。如果 Krita 並沒有自動使用正體中文介面語言，請參考以下步驟：

  1. 在上方功能表選擇 Settings ‣ Switch Application Language... 選單項目，然後會彈出一個小對話框。

  2. 點選 Primary language 並選擇「繁體中文」。

  3. 點選 OK 以關閉對話框。

  4. 關閉並重新開啟 Krita 後，Krita 便會以正體中文顯示。

如果以上步驟不能使 Krita 以正體中文顯示，你可能要再點選 Add Fall-back Language 加入一個「備選語言」並設定為「繁體中文」。這是一個程式錯誤，而我們仍未找到解決方法。

我在使用 Krita 時遇到了問題，如何可以獲得支援？¶

  1. 你可以到 Krita Artists 社群討論區發問（使用英文）。

  2. 請謹記，與你互動的人很有可能是志願工作者，而即使不是志願工作者也不是受聘提供使用者支援。請有善對待為你提供協助的人。他們會同樣地對待你。

  3. 若要提問，請為問題建立一個新話題（發文）並在標題中簡單描述一下問題。即使你想不到如何描述，只要標題中包含一些分類資訊，例如 "Krita's text tool is acting weird"（「Krita 的文字工具運作出現怪現象」），這已經比單純一句 "Help!" 來得更好了。

  4. 寫上你的問題。你可以加入畫面截圖和錄影，不過這樣做的話請同時用文字描述一下影片的內容和問題的重點。

  5. 盡可能列出有關係的資訊，最重要的是作業系統（Windows、macOS、Linux……），你使用的 Krita 版本編號（可到 說明 ‣ 關於 Krita 查看），以及你在使用的繪圖板類型、型號（如問題與數位板驅動程式相關）。

  6. 回答提供支援者的提問，即使你認為提問對解決問題沒有幫助。正在為你提供協助的人多數是已經有某些構想才會希望查詢相關資訊。

  7. 如果不知道如何回答，或者不了解如何能獲得有關資訊，應請求支援者進一步解說清楚。

  8. 如你回報了問題，在收到回應要求進行測試時請盡量抽空協助。這會有助加快找出問題根源並修正問題的過程。

Krita 有沒有「圖層剪裁」或「剪裁遮色片」的功能？¶

Krita 沒有「剪裁遮色片」，不過有圖層「繼承透明度」這個選項可以做出相同的功能。請參考這個關於「繼承透明度」的頁面。

Krita 的設定檔存放在哪裡？¶

它們的存放位置取決於作業系統，分別如下：

Linux

$HOME/.config/kritarc

Windows

%APPDATA%\Local\kritarc

macOS

$HOME/Library/Preferences/kritarc

kritarc 檔案為設定檔。Krita 並不會把設定寫入 Windows 登錄檔。

重設 Krita 的設定選項¶

你可以使用以下方法之一來重設 Krita 的設定：

  • 在 4.3 版本新加入: 你可以從使用者介面重設設定。在選單中點選 設定 ‣ 重設所有設定 後會出現一個確認對話框讓你確認重設操作。即使你選擇了重設設定，Krita 仍會將原先的 kritarc 檔案備份為 kritarc.backup。如你將此備份檔案重新命名為 kritarc，你便可將 Krita 的設定還原到在重設之前的狀態。此備份檔案可在下列位置找到：

    Linux

    $HOME/.config/kritarc.backup

    Windows

    %LOCALAPPDATA%\kritarc.backup

    macOS

    $HOME/Library/Preferences/kritarc.backup

  • 在啟動 Krita 時按住 Shift + Alt + Ctrl 組合鍵。這樣 Krita 會顯示一個對話框詢問你是否要重設 Krita 的設定選項。點選是以重設。

  • Krita 3.0 或之後版本：刪除在下列位置找到的 kritarc 檔案：

    Linux

    $HOME/.config/kritarc

    Windows

    %LOCALAPPDATA%\kritarc

    macOS

    $HOME/Library/Preferences/kritarc

你可能會找到這兩個檔案，你亦可考慮將它們移除：kritaopenglrc 和 kritadisplayrc。

如你的設定選項使 Krita 出現軟體崩潰，請將這個設定檔案備份，並將它傳送給 Krita 開發者以研究其導致崩潰的原因。

如你經由 Microsoft Store 安裝 Krita，kritarc 檔案可能會放在另一個位置：

%LOCALAPPDATA%\Packages\49800Krita_隨機字串\LocalCache\Local\kritarc

當中的「隨機字串」在每個安裝上都各有不同。

Windows 使用者留意，除非安裝後的應用程式檔案因其他程式（例如某些病毒防護軟體）或者硬碟故障而受到損壞，否則解除並重新安裝相同版本 Krita 是不能解決任何問題。重新安裝只會覆寫 Krita 執行檔和內置資料，並不會重設你的 Krita 設定選項。

為何 Krita 的設定選項會自動重設？¶

有兩個可能性：

  • 你沒有在修改設定後點選確定以儲存設定。

這有可能發生在解析度較低的螢幕上，或者是啟用了顯示器縮放的 Windows 系統上。Krita 的 設定 Krita… 視窗或許會過高，導致位於底部的 確定 按鈕超出顯示器的可見範圍。你可以嘗試使用 Alt + O 組合鍵來觸發確定按鈕。（到 設定 Krita… ‣ 一般 ‣ 視窗 可以停用 啟用高解析度螢幕 (Hi-DPI) 支援 選項以無視顯示器縮放來減少 Krita 介面佔用的空間。）

  • 你使用了電源按鈕來關機。

如果你沒有按正常程序關機或重新啟動電腦，Krita 的設定檔有可能會損壞。請按正常程序使用「開始功能表」中的關機選項。

我的 Krita 資源檔案存放在哪裡？¶

請參見資源管理一頁。

Krita 彈出對話框表示找不到某些檔案，然後自己關閉，應該如何解決？¶

可能是下列原因：

  • 有可能是你的下載檔案損壞了並缺少了某些檔案（通常在 Wi-Fi 或網路連線質素較差的情況下出現），在這些情況下你應嘗試使用較佳的網路連線來下載。 Krita 應用程式下載大小應為約 120 至 170 MB。

  • 有可能是安裝時出現問題。請檢查硬碟是否有足夠可用儲存空間：安裝 Krita 需要約 500 MB 的可用空間。如不是可用空間不足，你的電腦可能出現了其他問題，而我們建議找電腦專家協助查出問題和解決方法。

  • 某些程式可能不能正確解壓縮 ZIP 檔案。Windows、macOS 和大部份 Linux 發行版內置的解壓程式應可以正常運作，而我們建議採用這些程式。

  • 有可能是你曾手動移動或刪除了某些 Krita 的內置資源檔案，而導致 Krita 不能找到它們。

Krita 支援哪些顯示卡？¶

Krita can use OpenGL to accelerate painting and canvas zooming, rotation and panning. Nvidia and recent Intel GPUs give the best results. Make sure your OpenGL drivers support OpenGL 3.2 as the minimum. AMD/ATI GPU’s are known to be troublesome, especially with the proprietary drivers on Linux. However, it works perfectly with the Radeon free driver on Linux for supported AMD GPU. Try to get a graphics card that can support OpenGL 3.2 or above for the best results, some examples:

Intel

Intel 3rd Generation HD Graphics, IvyBridge or Bay-Trail microarchitecture, released in 2012. Commonly available products: Celeron J1x00, N2x00, Celeron (G)1xx0, Pentium J2x00, N3500, Pentium (G)2xx0, Core i3/5/7-3xx0.

AMD/ATI

Radeon HD 2000 family, TeraScale 1 microarchitecture, Released in 2007. Commonly available products: Radeon HD 2400 PRO, Radeon HD 2600 PRO, etc.

NVIDIA

GeForce 8 family, Tesla microarchitecture, released in 2006. Commonly available products: GeForce 8400 GS, GeForce 8800 GTS, 9800 GTX, GTS 250, etc.

Krita 3.3 或更新版本：Krita 在 Windows 上可以使用 Direct3D 11（經由 ANGLE）作圖形加速。在使用 Intel 顯示卡的系統上，這個選項會自動被選取。

我不能編輯由 Photoshop 建立的 PSD 檔案內的文字¶

Krita 現時未支援 PSD 檔案的文字格式。文字會以點陣圖形式轉為顏料圖層。

我的影像會佔用多少記憶體？¶

就基本影像而言，簡單來說可以這樣計算：寛度像素 乘以 高度像素 乘以 色彩通度數目 乘以 色彩通度位元組數（例如一張 1000×1000 像素的 16 位元整數 RGBA 影像：1000 × 1000 × 4 × 2）。這個數再乘以 圖層數目加二（一為影像本身，一為顯示用）。如果影像含有遮罩、濾鏡圖層或再製圖層，計算則變得複雜。

為何當我使用橡皮擦時，會出現棋盤方格圖案？¶

你可能習慣了 GIMP 或 Photoshop 的模式：它們預設的背景或第一個圖層預設沒有 Alpha（透明度）通道，因此在背景圖層使用橡皮擦工具時會填上背景色。

Krita 中所有圖層均帶有 Alpha 通道。如果你想保留背景色，你只需使用第一個（最底部）背景圖層以上的圖層來繪畫，這樣你在使用橡皮擦時便不會擦除背景色而令棋盤方格出現。大多數 Krita 使用者都會首先加入一個空白圖層才開始工作。（鍵盤上的 Ins 鍵可讓你方便地加入空白圖層。）這不會令影像秏用多餘的記憶體，因為含有預設背景色的空白圖層只會佔用一個像素的記憶體。

Krita 可以編輯 8 位元索引色彩影像嗎？¶

不能。Krita 由始至終都是以採用真實色彩而設計，而非索引色彩。我們沒有計畫加入對索引色彩的支援。即使 Krita 現可匯出某些索引色彩影像格式（例如 GIF），但在匯出時沒有提供控制調色板的功能，不能確保特定像素的色彩值。

在 Windows 上，Krita 的使用者介面在螢幕上的顯示過大¶

如果你使用 Windows，你可能會設定顯示器縮放為 150% 或 200%。Krita 預設啟用了 High-DPI 顯示器縮放，Krita 的介面或許會因此而變得太大，大於顯示器可顯示的空間。你可以依照下列步聚將其停用：

  • 在主選單上點選 設定 ‣ 設定 Krita…

  • 在 一般 版面中選取 視窗 分頁。

  • 取消勾選 啟用高解析度螢幕 (Hi-DPI) 支援（或者如果你想要啟用這項功能的話，請將其勾選）

  • 在 5.0 版本新加入: 如果你的顯示器縮放設定為非整數（即不是 100% 的倍數）的話，你也可以嘗試保持啟用 啟用高解析度螢幕 (Hi-DPI) 支援 選項並同時啟用 啟用非整數比例縮放 (Hi-DPI) 選項。

  • 點選 確定。如設定視窗顯示過大，你也可嘗試使用 Alt + O 鍵盤組合鍵來觸發確定按鈕。

  • 重新啟動 Krita

你也可以在工具箱上點按滑鼠右鍵以選取工具箱圖示的大小。

Windows：在全螢幕模式中，為何畫面最下方會有一條幼邊？¶

當畫布圖形加速繪製器設定為 OpenGL 時，你或許會在全螢幕 Krita 視窗的底部看到一道穿透的橫線。這是為了解決在全螢幕模式時不能使用選單和下拉方塊的問題而作出的折衷方案。如果這困擾著你，你可以考慮將繪製器改為 Direct3D 11，這沒有同樣的問題。

Windows：OBS 不能錄製 Krita 的 OpenGL 畫布¶

可以嘗試以下解決方法：

  1. 在 設定 ‣ 設定 Krita… ‣ 顯示 停用 OpenGL。

  2. 或在 OBS 中不要使用硬體加速模式（遊戲擷取模式），取而代之使用視窗擷取或顯示器擷取模式。

你或許也可以選取使用 ANGLE 繪製器而不使用原生 OpenGL。

我們不建議這樣做。Sandboxie 攔截資源檔案讀取的方式或會引致運作不順暢或停頓。

Windows：Krita 不能開啟我的檔案¶

你的文件出現損壞。發生這個情況有可能是因為：

  1. Windows 系統經歷了不正常關機或電源中斷，例如是你長按了電源按鈕來關機。這些情況可能會使系統未能正常完成將檔案寫入硬碟的程序。請謹記一定要依照正常的關機步驟來關上電腦。如情況不許可（例如你身處於經常會出現電力供應問題的地區），請每日定期備份你的檔案。不正常電源中斷有機會會使檔案內容被清空為零，發生這些情況的話是沒有可能復原檔案內容。

    在 4.2.8 版本變更: Krita 4.2.8 版本中引入了安全措施有助避免這個情況發生。不過無論如何，除非出現無法控制的情況，請一定要依照正常的關機步驟來關上電腦。在 Windows 下你應該使用「開始功能表」中的關機選項。

  2. 不良品質的防毒軟體可能會試圖修改 KRA 檔案或防止 Krita 正常將檔案寫入到目標資料夾中。要檢查這種情況，你可以先嘗試從 Krita 將檔案儲存到資料夾位置，然後在不關上 Krita 的情況下到該資料夾檢查檔案有沒有被正常儲存。因這種情況丟失的檔案不能被還原。

  3. 雲端儲存空間如 Dropbox 和 OneDrive 的用戶端同步功能已知有可能會阻礙 Krita 正常儲存檔案。雖然我們已為這種情況加入了修正方案，但是也值得自行測試一下這些同步功能有沒有在產生相關問題。因這種情況丟失的檔案不能被還原。

  4. 有時候，構成 KRA 檔案格式的 ZIP 檔案會缺少了檔案的最後數個位元組。我們已盡力採取可行的措施來預防這類檔案損壞問題，但這也有可能是磁碟區檔案系統的問題。這種情況可以將檔案的副檔名重新命名為 ZIP（需先在 Windows 中啟用顯示副檔名，方法可自行在網上搜尋），然後使用一些能修復 ZIP 壓縮檔的軟體來修復這個檔案，再將經修復後的檔案改回副檔名為 KRA。

  5. 如 Krita 沒有顯示錯誤訊息，而是出現軟體當機或崩潰的情況，這很可能代表你的影像太大了，系統沒有足夠的記憶體讓 Krita 處理該檔案而導致作業系統強制終止 Krita 運行。請嘗試關閉其他軟體（例如網頁瀏覽器或影視串流服務）以騰出可用記憶體，然後再嘗試開啟該檔案。一般建議的後續行動是盡量縮減該檔案的大小，例如是將圖層合併、分割動畫檔或把影像尺寸縮小。

如何復原我的檔案？¶

  1. 檢查有沒有任何備份檔案或自動儲存備份：儲存、自動儲存及備份檔案。

  2. 嘗試能不能以 ZIP 檔案格式開啟檔案。

    1. 將副檔名由 .kra 改為 .zip。

    2. 嘗試將其開啟（系統應該會自動選用壓縮檔工具開啟）。

    3. 壓縮檔內應該含有一個名為 mergedimage.png 的檔案，儲存了合併圖層後的影像。如無法復原整份文件，你仍可以這張影像作參考。

  3. 嘗試使用修復 ZIP 壓縮檔的軟體。

    1. 為檔案建立副本，留作備份以備不時之需。

    2. 將副檔名由 .kra 改為 .zip。

    3. 用 ZIP 壓縮檔修復軟體修復 .zip 檔。

      # 在 Linux：
mv file.kra file_copy.zip
zip -F file_copy.zip --out file_new1.zip
unzip file_new1.zip
# 如果這還不行的話：
zip -FF file_copy.zip --out file_new2.zip
unzip file_new2.zip
# 如果這還不行的話，嘗試以 *file_new2.zip* 檔再運行一次，或嘗試以 *file_new_1.zip* 檔運行。

# 在 Windows：
複製檔案然後修改其副檔名。
使用修復 ZIP 壓縮檔的軟體。（請依照該軟體所提供的指示。）

    4. 嘗試在 Krita 內開啟。

    5. 如不能在 Krita 開啟該檔案，請嘗試步驟 2. 的措施：開啟壓縮檔並在裡面找出 mergedimage.png 檔案。

  4. 以 Notepad（記事本）或任何純文字編輯器開啟檔案。如檔案內容只含有重覆的 NULL 符號，這表示該檔案很大機會無法以一般方式還原。如該文件非常重要，你或可嘗試使用會掃描整個硬碟的方法來還原先前的存檔。

Krita 在 Windows 7 上啟動時當機而強制終止¶

自 Krita 4.2.0 起，Krita 使用了 Qt 5.12 程式庫。這需要 Direct3D 11 或 OpenGL ES 2.0 圖形加速。你需要安裝適當的顯示卡驅動程式（NVidia、AMD/ATI、Intel）。這也表示 Krita 在虛擬化系統上運行比較困難：使用 VirtualBox 時你需要在安全模式內安裝 Guest Additions，同時啟用實驗性的 Direct3D 支援。

Krita 在 Windows 上運行不時會出現卡住的情況¶

Are you using a dictionary app (e.g. Youdao Dictionary for Chinese users)? Some dictionary apps can read words from other app's windows and show popup translations in real time. However, it has been reported that such apps tend to cause Krita to freeze randomly. If you are using one of those, make sure to QUIT them (no notification icon) when using Krita. Some of those apps keep running in the background even after being closed. In such case, you will have to uninstall them.

Windows：Krita 的視窗顯示為半透明¶

Chances are you are using an NVIDIA GPU. Due to a bug in NVIDIA's driver that we haven't been able to work around yet, sometimes Krita's window will be transparent or semi-transparent. The solution is to enable the Angle renderer in Krita's Settings dialog. Open the Settings menu (Press Alt-N if the menubar is not visible and your system is in English), then open the Configure Krita dialog. In the dialog window select the Display page and select the Angle renderer in the Preferred Renderer drop down. Restart Krita.

為何某些工具面板的標題列上會出現「&」符號？¶

這是我們使用的某第三方程式庫中的一個 bug，未能於 Krita 直接修正。「&」符號是用來標記選單捷徑字元的前綴，正常會使隨後的字元以底線標示。

這個 bug 只在特定的系統配置上出現（與「Fusion」樣式有關）又或者是從第三方所取得的 Krita 軟體包（例如部分 Linux 發行版）。

如果你正在使用 Linux，最好的做法是到 krita.org 取得官方的軟體包，例如是在下載頁面中提供的 AppImage。

數位繪圖板¶

Krita 支援哪些繪圖板？¶

Krita 作為數位繪畫軟體，當然要使用帶有筆壓感應的數位繪圖板才能物盡其用。如果數位板已經正確設定，Krita 應該能立即使用。

在 Windows 上，你需要安裝該數位板專用的 WinTab 驅動程式，或在 Krita 設定中啟用 Windows 8+ 指標輸入 (Windows Ink)選項。

你可以在連結頁面中找到由社群貢獻整理的支援數位板清單。

如果你在找尋與平板電腦（如 Android 或 iPad）相關的資訊，請參見連結頁面。

Krita 不能識別我的繪圖板，可以怎樣做？¶

首先檢查一下你有沒有安裝數位板的驅動程式。數位繪圖板一頁描述並解釋了一些常見的問題。如這些資訊沒有幫助，我們希望你到 bugs.kde.org 回報問題並附上數位板紀錄檔。以下是獲取數位板紀錄檔的步驟：

  1. 你需要使用特定工具來檢視紀錄檔輪出。由 Krita 4.2 起你可以使用紀錄檢視器工具面板，在開啟面板後啟用記錄。

    在 4.2 版本變更: 紀錄檢視器為 Krita 4.2 版本中新增的功能。如使用較舊版本的 Krita，你需要以終端機啟動 Krita（適用於 Linux 或 macOS），或從微軟的網站安裝 DebugView 程式然後再啟動 Krita（適用於 Windows）。

    使用終端機時，請啟用無限往回捲動。

  2. 按下鍵盤快捷鍵 Ctrl + Shift + T，便會出現一個對話方塊告知你已開始進行記錄。

  3. 嘗試重現你發現的問題。在使用數位板繪畫的同時，你應能夠看到紀錄檢視器中出現新的紀錄項目。

  4. Save the output from the log viewer into a TXT file, and attach it to the bug report.

On Linux, it is also useful to have the following information:

  1. lsmod

  2. xinput

  3. xinput list-props (ID can be fetched from the item 2)

However, in 100% of the cases where Windows users have reported that their tablet didn't work over the past five years, the problem has been either a buggy driver or a broken driver installation, but not a bug in Krita.

如何修正繪圖板筆尖與游標位置不符合的問題¶

If you see that your tablet pointer has an offset when working with Krita canvas, it might be highly likely that Krita got an incorrect screen resolution from the system. That problem happens mostly when an external monitor is present and when either a monitor or a tablet was connected after the system booted.

You can configure this by going to the 「數位板」設定.

Microsoft Surface Pro 及 N-Trig¶

Krita 3.3.0 and later supports the Windows Pointer API (Windows Ink) natively. Your Surface Pro or other N-Trig enabled pen tablet should work out of the box with Krita after you enable Windows Ink in Settings ‣ Configure Krita... ‣ Tablet.

Tablet Pro 與 Surface Pro¶

Unlike Wacom's Companion, the Surface line of tablets doesn't have working hardware buttons. Tablet Pro is a (non-free) utility that puts virtual buttons on screen. Krita 3.1 and above will have predefined shortcut profiles to work with Tablet Pro.

https://tabletpro.com/

See https://www.youtube.com/watch?v=WKXZgYqC3tI for instructions.

在 Windows 上使用繪圖筆時出現筆觸動作、白色圓圈、彎曲線條等等的奇怪現象¶

如何設定我的最愛筆刷預設？¶

Right-click a brush in the brush docker and assign it a tag. Then right-click on canvas to call popup palette, click the second right-most icon on the bottom-right of the palette, now you can pick the tag which contains the brush you assigned to it.

Krita 可以載入 Photoshop 的筆刷嗎？¶

Yes, but there are limitations. You can load ABR files by using the Import button in the Predefined brush tab in the brush editor. Since Adobe hasn’t disclosed the file format specification, we depend on reverse-engineering to figure out what to load, and currently that’s limited to basic features.

Krita 運行得不𣈱順¶

There is a myriad of reasons why this might be. Below is a short checklist.

  • Something else is hogging the CPU or the memory: Spotify and other Electron apps have been known to do this.

  • You are running Windows, and have 3rdparty security software like Sandboxie or Total Defender installed

  • You are working on images that are too big for your hardware (dimensions, channel depth or number of layers)

  • You do not have canvas acceleration enabled

  • You have (NVIDIA) Vertical Sync enabled

  • On macOS, with some macs, you might need to disable canvas acceleration in Krita's settings.

Please also check this page.

啟動時間過長¶

You probably have too many resources installed. Deactivate some bundles under the Settings ‣ Manage Resource Libraries... menu item.

If you're using Windows with the portable ZIP file, Windows will scan all files every time you start Krita. That takes ages. Either use the installer or tell Microsoft Security Essentials to make an exception for Krita.

筆刷繪製緩慢¶

  • Check if you accidentally turned on the stabilizer in the tool options docker.

  • Try another scaling mode like trilinear. Settings ‣ Configure Krita... ‣ Display.

  • Try a lower channel depth than 16-bit.

  • For NVIDIA, try a 16-bit floating point color space.

  • For older AMD CPU's (Krita 2.9.10 and above), turn off the vector optimizations that are broken on AMD CPUs. Settings ‣ Configure Krita... ‣ Performance. This isn't needed if you've got an AMD Threadripper™ CPU.

  • It's a fairly memory-hungry program, so 2 GB of RAM is the minimum, and 4 GB is the preferable minimum.

  • Check that nothing else is hogging your CPU.

  • Check that Instant Preview is enabled if you're using bigger brushes (but for very small brushes, make sure is disabled).

  • Set brush precision to 3 or auto.

  • Use a larger value for brush spacing.

  • If all of these fail, record a video and post a link and description on the Krita Artists Forum in appropriate category.

  • Check whether OpenGL is enabled, and if it isn't, enable it. If it is enabled, and you are on Windows, try the Angle renderer. Or disable it.

在使用一段時間後效能變得緩慢¶

Once you have the slowdown, click on the image-dimensions in the status bar. It will tell you how much RAM Krita is using, if it has hit the limit, or whether it has started swapping. Swapping can slow down a program a lot, so either work on smaller images or turn up the maximum amount of RAM in Settings ‣ Configure Krita... ‣ Performance ‣ Advanced Tab.

為何產生的動畫在影片播放程式中只顯示黑畫面？¶

在使用一段時間後某些快捷鍵失效¶

Have you loaded any Keyboard or Canvas Shortcut Schemes (e.g. Photoshop/SAI compatible schemes) other than the Default one?
